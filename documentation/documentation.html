Frame Interpolation and Video Upscaling/Enhancement Project

By Roberto Tallarini, Lorenzo Gerosa & Gianluca Mariani

Processing Steps:

  • Motion Frame Interpolation
    1. demo.py passes the video path to the frame interpolation function.
    2. The video interpolation function increases the frame count by calculating optical flow between consecutive frames both forward and backward, and then interpolates N new frames between each pair of frames.
    3. The original frames are discarded.
    4. The video is saved.
  • Video Upscaling
    1. demo.py calls the video upscaling function, passing the video path. For each frame, the function invokes the bilinear upscaling function.
    2. Once a frame is upscaled, image enhancement filters are applied, and the frame is saved in the new video until completion.
  • End of the System

Legend:

"Interpolation First" Switch:

Allows choosing whether to perform frame interpolation or upscaling first. By default, upscaling is performed first. From a computational perspective, it is more resource-intensive to upscale first and then interpolate the frame rate, but it yields better results.

Zoom Factor:

A decimal number defining the scaling factor.

Upscaling Iterations:

An integer indicating the number of upscaling iterations to perform. The resolution after upscaling is the initial resolution * (zoom factor upscaling iterations).

Number of Frames to Interpolate:

An integer indicating the number of frames to interpolate between each pair of frames (original frames are discarded). The final frames per second after interpolation is the number of frames to interpolate * (initial frame count - 1).

Filters

During upscaling, enhancements are applied, including Gaussian blurring, Laplacian edge detection, and sharpening. All these filters are considered as a single filter in the graphical interface and applied sequentially according to an optimal algorithm. However, parameter customization is possible.

Select a Default Filter:

Allows choosing from a set of default filters written for specific videos during testing.

Customize the Filter:

Allows customizing the parameters of the mentioned filters.

Show Edges:

Keep this option disabled. In testing, it allows enabling or disabling the display of overlaid edge detection on the image.

Filter Parameters:

Gaussian Blurring Kernel Size:

This parameter represents the size of the kernel used to apply the blurring filter in the initial stage of upscaling. The kernel size influences the degree of blurring applied to the image. Larger values generate more intense blurring.

Gaussian Blurring Sigma X:

Sigma X is a parameter of the blurring filter that affects the standard deviation of the Gaussian used for blurring. Higher Sigma X values generate more diffuse blurring, while lower values create more localized blurring. It is important that the sigma value is proportional to the kernel size.

Sharpening Kernel Central Value:

This parameter represents the central value of the kernel used to enhance image sharpness during upscaling. Increasing or decreasing the central value influences the intensity of the applied sharpness boost. Every other kernel value is calculated with equal intensity to always result in a sum of 1.

Laplacian Kernel Size:

The size of the Laplacian kernel affects image sharpness and can be adjusted to achieve more or less intense edge detection effects.

Binarization Threshold:

This parameter controls the threshold value used during the image binarization process. Binarization is a process that converts the image into binary form, where pixels above the threshold are considered edges. Adjusting the threshold value determines how many edges are included in the final image.

Median Blurring Kernel Size for Denoising:

This parameter represents the size of the kernel used to apply the median blurring filter during frame interpolation to remove impulse noise, salt, and pepper noise.
